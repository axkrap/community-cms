comcms_blocks

Description:

This table keeps a list of all configured blocks, and their properties.

Fields:

id: A unique positive integer used to identify the block.
type: String identifying which data files to use for the block. Block types are defined in content_blocks/blocks.info.
attributes: String containing comma separaged attributes in the format attb1=value,attb2=value2.

comcms_config

Description:

This table holds the site-wide configuration that may change over time. Database information is not variable: it almost never changes. On the other hand, the site footer may change often, so it should be stored in the database. This means you can remove write permissions from config.php after Community CMS is installed.
